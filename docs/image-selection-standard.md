# Zboystyle 选图标准

这份标准用于约束首页图、栏目图、文章图，避免以后再次出现“图能用但气质不对”“技术没错但看起来很怪”的问题。

## 一、总原则

Zboystyle 是青年男装 / cleanfit / softboy / 男大 / 韩系日常风格导向的网站。

所以图片优先级必须是：
1. **真人上身是否成立**
2. **是不是中国青年潮男 / 东亚青年男装语境**
3. **是否能直接支持文章或页面主题**
4. **是否足够时尚、干净、可浏览**

不是“图好看就行”，而是“图是否属于这个站”。

## 二、首页 Hero 图标准

首页 Hero 图必须满足：
- 必须是 **真人上身图**，不能只是包、鞋、静物、风景
- 最好能直接看出青年男装、cleanfit、校园感、轻韩系、软少年感等气质
- 优先选择年轻、干净、顺眼、真实可模仿的男装穿搭图
- 最好一眼看上去就让人知道这是“潮男 / 青年男装网站”

### 首页 Hero 禁用图
- 老年男性或明显年龄气质不符的模特
- 熊、动物、风景、街景空镜、建筑图
- 只有单品静物、没有人物上身的图
- 太像复古档案照、博物馆照、品牌老广告的图
- 过于正式商务、成功学男装、老派绅士风图
- 过于欧美化且与站点目标语境明显脱节的图

### 首页 Hero 理想图关键词
- 中国潮男
- cleanfit 男生
- 男大穿搭
- 韩系青年男装
- 校园感穿搭
- 轻通勤年轻男装
- 低饱和、干净、顺眼、带一点层次

## 三、栏目图 / 卡片缩略图标准

栏目页和首页图文卡片图必须满足：
- 一眼能看出是这个栏目相关内容
- 优先真人上身图，其次才是单品图
- 小图条件下仍然能认出主体是什么
- 不要选构图太空、主体太远、信息太散的图

### Tops / 上衣类
优先：
- 真人上身
- 能看清领口、肩线、版型、长度
- 最好能看出整套 cleanfit / 男大语境

避免：
- 只有布料特写
- 远景环境图
- 看不出到底穿了什么

### Bottoms / 下装类
优先：
- 能看清裤型、长度、垂感、鞋裤关系
- 最好包含下半身完整比例

避免：
- 上半身过重、裤子反而不清楚

### Accessories / 配饰类
优先：
- 配饰在真实穿搭中的使用状态
- 包、帽、眼镜、项链与整体搭配关系清晰

避免：
- 纯静物棚拍感过强，像电商白底图

### Style / 风格类
优先：
- 一张图就能表达这个风格的轮廓和气质
- 有完整穿搭关系，而不是只看到单件衣服

避免：
- 只有局部特写
- 没有风格信息的普通人像

## 四、文章正文图标准

文章正文图的任务是“帮助阅读理解”，不是“单纯撑场面”。

### 优先级
1. 真人上身图
2. 能支撑段落观点的细节图
3. 对比图 / 并排图

### 禁止做法
- 风景图凑数
- 熊图/动物图凑数
- 完全跑题的 lifestyle 图凑数
- 只有静物却硬说在讲穿搭效果
- 一张过高的图占满屏幕，不给正文留空间

### 正文图版式规则
- 高图 / 竖图：优先收窄，必要时文字环绕
- 宽图：可以放宽，但不能打断阅读节奏
- 同一论点下多张图：优先考虑并排排版
- 细节图不能全篇到处堆，必须和上身图搭配

## 五、图片气质判断（快速人工检查）

发布前快速问这几个问题：
- 这张图一眼看上去像不像青年潮男网站会用的图？
- 如果用户只看图，不看标题，会不会误会这是别的网站？
- 这张图是不是“虽然能用，但明显不够对味”？
- 这张图有没有让人想问：为什么是这个？

如果会触发下面这种反应，就说明图错了：
- 为什么是老头？
- 为什么是熊？
- 为什么是风景？
- 为什么像旅游、博物馆或复古档案网站？
- 为什么像 generic 图库，而不是潮男内容站？

## 六、发布流程建议

每次上图至少过这 4 步：
1. 先判断“是不是人、是不是上身、是不是对题”
2. 再判断“缩略图状态下是否仍能看懂”
3. 再判断“会不会把正文排版冲垮”
4. 最后判断“是否符合 Zboystyle 的整体气质”

## 七、当前执行策略

在没有新外部图源接入前：
- 现有站内图片里优先使用最接近青年男装真人上身的图
- 把明显跑偏的图逐步替换掉
- 后续补进新的中国潮男 / cleanfit / 男大 / 韩系青年男装图片库后，再做第二轮替换
