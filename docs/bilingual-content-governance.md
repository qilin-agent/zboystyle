# Bilingual Content Governance

本文件只描述 `BoyStyle` 当前双语内容的治理规则。

## 1. 中文是源稿

`BoyStyle` 的双语内容体系里，中文稿是源稿。

默认流程是：
1. 先完成中文稿
2. 确认中文稿结构、事实、判断与结论成立
3. 再生成英文稿
4. 再做中英对齐检查
5. 最后发布

禁止先写英文再倒推中文。

## 2. 英文必须建立在中文稿之上

英文稿必须翻译、重写并对齐自对应中文稿。

允许：
- 为英语读者补足背景解释
- 为英语表达做自然重写
- 对中文互联网特有概念做更明确解释

不允许：
- 脱离中文稿单独起题
- 脱离中文稿单独立意
- 把英文稿写成另一篇重点不同的文章
- 改变中文稿的核心事实、主线、判断或结论

## 3. 中文未定稿，不发布英文稿

如果中文稿尚未定稿，则英文稿不应先发布。

包括但不限于以下情形：
- 中文标题仍在漂移
- 中文结构尚未稳定
- 中文核心判断尚未稳定
- 中文稿仍在补重要信息

## 4. 中英文必须对齐的层面

发布前必须确认中英文在以下层面对齐：
- 页面主题
- 核心事实
- 结构主线
- 关键判断
- 最终结论
- section
- permalink
- `lang_switch_url`
- title / description 主旨

允许不同语言表达方式不同，但不允许内容意义跑偏。

## 5. 英文是镜像稿，不是平行独立稿

英文稿的任务是把中文稿转换为对英语读者可读、可理解、可进入的版本。

它不是：
- 另一个编辑视角的平行稿
- 借同一主题另写一篇新文章
- 看起来对应、实则重点不同的“伪镜像稿”

## 6. 标题与摘要规则

### 6.1 标题主旨要对齐
中英文标题不要求逐词直译，但必须表达同一个核心切角。

### 6.2 description 主旨要对齐
中英文 description 可以有语言自然化差异，但不能一个强调趋势，一个强调店铺，或一个强调风格判断，另一个强调购买指南。

## 7. metadata 对齐规则

每对双语页面发布前至少检查：
- `section`
- `permalink`
- `lang`
- `lang_switch_url`
- `title`
- `description`
- `date`
- `updated`
- `featured`
- `index_title`
- `index_description`
- `thumbnail_image`
- `thumbnail_alt`

metadata 可以做语言适配，但不能失配。

## 8. 更新时的动作规则

如果中文稿更新了以下内容，英文稿必须同步检查：
- 标题
- 结构
- 新增段落
- 删除段落
- 核心判断
- 结论
- metadata
- 图片说明

不能只改中文、不看英文镜像稿。

## 9. 禁止的常见错误

禁止出现：
- 中文在写风格判断，英文在写店铺导购
- 中文判断克制，英文包装夸张
- 中文更新后，英文仍是旧版本
- `lang_switch_url` 正确，但内容已不再成对
- 中英标题看似对应，实际切角已不同

## 10. 发布前最小检查清单

每对双语页面发布前，至少确认：
1. 中文稿已定稿
2. 英文稿源自中文稿
3. 中英标题主旨一致
4. 中英 description 主旨一致
5. 中英结构主线一致
6. 中英结论一致
7. `lang_switch_url` 正常
8. permalink 对齐
9. metadata 对齐
10. 已完成 commit 与 push
